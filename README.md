## 커밋 메세지
1. 메세지 구조
<type>(<scope>): <subject> -- 헤더 
<BLANK LINE> -- 빈 줄 
<body> -- 본문 
<BLANK LINE> -- 빈 줄 
<footer> -- 바닥 글

출처: https://xtring-dev.tistory.com/entry/Git-규칙적인-Commit-메세지로-개발팀-협업하기-👾 [xtring.dev]
2. 메세지 규칙
  * 제목과 본문을 행으로 구분
  * 제목은 50자 이내로 작성
  * 제목의 첫 글자는 대문자
  * 제목의 끝에는 마침표를 사용하지 않음
  * 본문의 각 행은 72자 이내
  * '어떻게'보다 '무엇', '왜'를 설명

3. Type
  * feat : 새로운 기능에 대한 커밋
  * refactor : 리팩토링에 대한 커밋
  * test : 테스트 코드 수정에 대한 커밋
  * docs : 문서 수정에 대한 커밋
  * style : 코드 포맷, (;) 누락 등에 대한 커밋
